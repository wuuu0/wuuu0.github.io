前端学习日志

发表于 2021-09-09 更新于 2021-11-30 分类于 Front-end

该日志汇总了我收集的前端学习路线和学习资料，并简单记录一下我的前端学习过程。关于前者，明确的目的让你少做无用功，科学的步骤可以让你保持合适的节奏，有效的方法能则提高你的效率，时常铭记 “明确目的，保持节奏，提高效率”；至于后者，是因为学得再多，不记录不练习/使用总是会忘得很快，此外，也可以借此日志来督促自己的学习进度。

学习路线

· Luke 的前端路线

主要遵循Luke大佬给出的 前端学习路线 。此路线是他基于Roadmap to becoming a web developer in 2021 制作的，该路线图展示了你在学习前端开发过程中，你应该去学习的技术，并且作者也给出了他的建议，即哪些是他推荐去学的/哪些是不推荐或过时的，总结的非常详细。但过于详细的路线容易让新手陷入迷茫，不知道主路为何，不知道先学什么，后学什么，学到什么程度等等。

于是 Luke 整理了一个更适合新手入门的前端路径图。大佬 讲解学习路线的视频也非常不错，解释了如何使用这个线路图，也给出了清晰的学习节奏。

· 企业招聘要求 & 笔试面经

这部分内容与上一点的关系，就是 考纲 & 考试重点 和 课程学习 的关系。

前端牛客网面经：牛客网 前端面经，前端工程师面试宝典，「2021」前端面试题汇总，前端面试每日 3+1

学习日志

阶段一

浏览器原理

  • Luke的视频：浏览器是如何运作的？

    详细讲解了 从打开窗口输入 url 地址，到呈现出页面这个过程中，浏览器内部发生了什么事情。将浏览器的各个程序模块和初次加载流程讲的很清楚。
    不足之处在于，没有讲清楚 JS 在初次加载后的执行情景，没有讲到屏幕每帧刷新时浏览器会做什么。

    初次解析渲染完 html 后，JS 修改页面（代码修改dom和cssom），和重绘重排是怎么个对应关系？JS 每改变一处地方，就重新渲染一次？还是取决于屏幕刷新率？

  • 红宝书第二章，介绍script标签

    defer和async script标签，到底何时执行（书里面说，defer的话，在html解析完成后执行，又说在DomContentLoad事件前执行，这不是矛盾吗）

    还说异步加载的 script标签 最好不要（直接）修改页面结构，但其实是可以修改的，只是可能需要重新渲染（document.write 需要重新解析构造DOM，而document.body.appenChild 这种直接操作DOM的是否会好一些？）

  • JavaScript 权威指南 - 15.1章节 Web编程基础。

    较详细地讲解了 JS 在浏览器中的执行情景。

Html + CSS + JavaScript

2021.4.1 - 2021.8.22

  • （理论）HTML CSS 菜鸟教程，简单过了一一遍
  • （理论）JavaScript 红宝书比较详细地过了一遍
  • （理论）JS 运行时环境 - V8 /浏览器/Node

    学习一个语言，除了语法，还要理解它的工作原理。语言工作原理主要考虑 编译/解释时 和 运行时 （维基百科）

    V8 和 运行时 的大致区分，可见 Javascript工作原理引擎，运行时，函数调用栈。总结来说，V8 主要负责以下工作：编译和执行 JS 代码、处理调用栈、内存分配、垃圾回收。其他支持 JS 执行的部分则属于更大范围的运行时（浏览器/Node）。
    V8：Luke - V8引擎编译优化JS
    综合：How JavaScript works、JavaScript 工作原理（前链接中文版）、JS 工作原理、浏览器与Node的事件循环有何区别?

2021.9.1 - 2021.9.7

  • （实践）HMTL+CSS+JS 实战：DOM编程艺术代码敲了一遍

    改书最新版出版于2011年，有些东西还是比较老旧，有非常多的JS代码所实现的功能其实用 HTML5 和 CSS3 能更简单方便的实现。整本书就是开发一个比较简单的网页，包括写 html css js。
    初步体验了开发网站的基本流程，其中 令我印象深刻的是：
    JS的事先错误检查（是否支持dom的某个方法，dom是否有某个元素）、HTML CSS JS 的分离、一种模式：script标签内的脚本仅负责编写一些用于调用的函数，给window注册load事件的监听器- window的load事件发生，创建某些元素，给某些元素注册事件监听器 - 事件发生，触发相应的处理程序。
    此外就是熟悉了 HTML CSS JS 的语法。

Webpack（浅学）

2021.9 - 11.28

  • Webpack官方文档

    只看到 “开发” 章节

  • 3h Webpack从原理到实战-深入浅出

    侧重 Webpack 原理，重点讲解了 JS 模块化历史和 Webpack 的关系，webpack 的打包机制（即源文件是什么，中间经过了什么，打包后输出的静态资源是什么内容，它如何体现源文件的本意又能在old的浏览器中正常运行）。

    实战内容：.js文件中引入js模块、loader处理css模块并加载到网页显示、loader处理.jsx和ES6语法、plugin处理html

    视频中用的是webpack4，可能有些命令和配置不一样，导致最后没办法完全按教程复现出来。主要了解基本概念和webpack的动机，webpack核心概念以及配置“在干什么”和“大概是什么流程和逻辑”。这样的话，即使版本改变，也能较快跟进新版本。

  • 尚硅谷 最新版Webpack5实战教程——webpack简介

    了解 Webpack 的动机和作用。总结起来：实现各种资源的模块化开发、使用新特性提高效率保证安全性、实时监听开发过程使用热更新、项目结果打包压缩优化

  • 千锋最新前端webpack5全套教程

    学习webpack核心概念和配置使用，暂时看 1.2.1 - 2.1.4。主要内容：基本配置、自动引入资源、搭建开发环境、资源模块/管理（图片/字体、CSS/SASS、数据、JS）、代码分离（打包后分离 多入口/多个不存在引入关系的JS模块、原模块和引用的模块、公共代码的抽离；动态导入及其应用）、缓存、拆分开发/生产环境配置、source-map（打包文件和源文件的映射，用于调试）、devserver（模拟从服务器加载页面到本地浏览器运行）、模块热替换与热加载（模块 增换删和修改时，不必手动刷新重新加载整个页面，即自动和页面部分刷新，方便调试）

    该教程思路清晰，内容详实，webpack 版本也是最新的，知识点学习和实践操作都很顺畅。非常不戳。

  • 从 Web 开发的大背景 思考前端

    感觉有必要先了解 Web网页的开发模式 这一大背景：
    目前我对前端的认知是，用户在浏览器中输入URL，浏览器从服务器加载 HTML、CSS、JS 等资源，本地浏览器对获取的内容进行渲染显示，并响应用户对页面的操作。前端就负责整个页面的实现（所谓 HTML-结构内容 CSS-样式布局 JS-行为）+科学的代码风格和设计模式/理念，以及考虑在浏览器端和网络传输方面能做的性能优化，也可以了解服务端的开发使得我们写出对后台更友好的页面（甚至迈向全栈）——值得思考的问题是：什么情况下，浏览器要和服务器通信（请求数据，更新页面，打开新页面等）？又有哪些事情需要在服务端处理/哪些是从服务器获取数据后在浏览器处理？前端开发 和 MVC MVP MVVM 等 Web开发模式有什么关系（感觉更像是服务端而非前端的东西）。
