CSS clearfix清除浮动 - 简书
css 浮动和清除浮动 - 博客园
一般而言，需要浮动的元素，其父级元素需要清除浮动。若父元素（仅父，而非祖宗）本身是浮动的则不需要清除浮动。

设计样式的两种思路：预定义样式，哪个元素需要就给他添加预定义样式的类；针对特定的元素，给他另设样式。

导航 & 首屏内容

  • 按住alt + 点击光标，可同时选中多处进行编辑
  • 注意踩坑，CSS选择器中，给某个标签限定类别应该是：label.cls 不能留空格 label .cls
  • 导航栏淘仔的头部，有关定位的一些笔记
  • 设计首屏内容时，由于没有明显的分行，所以决定采用 box + float 的布局方式。即大致划分左右区域猴，单个区域内的每个模块块之间都是平级，由 float 在区域内浮动来实现布局。具体的步骤是：先写好各个块的html并给类别来进行标识，在css中严格设置各个块的大小，然后通过float来布局，最后通过每个模块的margin来微调。各大模块的位置布局好后，再进入到每个模块内部，进行细节的实现。
  • 图片画廊下的选择小圆点的实现。一个 div 包裹多个 span，然后将 span 设为 inline-block，而非 设置多个块级元素再布局。

  • 注意一些内联元素的排列，直接连续排列，不要回车隔开，因为浏览器会将内联元素间的换行解析为一个空格。硬要将内联元素换行，还希望内联元素之间没有空格的话，可以在父级设置 font-size = 0。

  • imgBox2 内的图片，用弹性盒模型来进行对齐布局。参考资料：、。

  • 注意一些样式的继承关系对效果的影响。比如 <a><img></a>，要调整图片的大小，应该调整img标签而非a标签。<div><a>\xxxx</div>，要改变xxxx字体颜色要修改a标签，改字体可以改div标签进而继承到a标签中。参考资料：。

  • 网页中的小图标可能会集中放在一个图片文件中。一来可以减少浏览器加载页面时对服务器的请求次数，二来用CSS插入各个小图标的时候可以同过 引用相同的图片但使用不同的定位 来实现。

  • text-align 属性。可以对子元素进行对齐，但似乎对块级元素无效。

  • 所有HTML元素都可以被看作盒子 。行内元素、块元素：、

  • HTML元素，有的主要是当“容器”（多是块元素，div、p、ul、li等），有的是直接带有“内容”（多是行内元素，span、a、img等）；而且有的是通过父级元素设置样式（子元素继承）来控制内容表现（p元素的font、color、align等），有的则是对内容直接对应的元素设置样式来控制内容表现（a元素的color、img的width、iconfont的font等）。这个点容易搞混。

  • 插入的 iconfont 小图标，需要单独设置样式，父级的字体样式不会继承下去。且 iconfont 小图标的垂直居中，要对iconfont用 verticle-align: middle 或 line-height。
