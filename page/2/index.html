头部搜索

  • 对float的理解：要意识到，正常情况下块元素独占一行。运用了float后（称为floatx），连续的float的元素会挤到同一行中去，紧随其后的块元素（称为div0）在划分块区域时会“忽视”float元素，但是块中的内容会被float的元素挤开（首先按默认情况布局，遇到floatx，则被即挤开）。div0后继续接块元素（称为div1），则会与div0分开，另起一行。其他资料：菜鸟教程 CSS Float定位

如果 div0 中的内容被 floatx 挤得放不下了，那 div0 也会相应的有一些变化？会。div0 的划分区域仍旧不变，但是内容会被挤出去，甚至会被挤出 div0 的区域，即溢出 overflow。而 div0 溢出的内容会在哪？
连续的 floatx 会如何排列？

左侧和右侧蓝色框住的块元素分别左浮动和右浮动，其后紧接的块元素 div.search 占用的块区域“忽视了”这两个块的存在。

  • 在处理右侧浮动的二维码时，对CSS定位有了更准确的理解。理解定位和浮动的两个重要概念是 文档流和实际占用位置。所谓“脱离文档流”，就是元素不按照其在html文档中出现的顺序/位置进行绘制显示。其他资料：菜鸟教程 CSS Position定位
  • CSS line-height 和 height 的关系。总之，height 决定块内容的总高度，而line-height是内容总高度中的一部分，且它决定了文本的每一行撑起的高度。

导航 & 首屏内容

  • 按住alt + 点击光标，可同时选中多处进行编辑
  • 注意踩坑，CSS选择器中，给某个标签限定类别应该是：label.cls 不能留空格 label .cls
  • 导航栏淘仔的头部，有关定位的一些笔记
  • 设计首屏内容时，由于没有明显的分行，所以决定采用 box + float 的布局方式。即大致划分左右区域猴，单个区域内的每个模块块之间都是平级，由 float 在区域内浮动来实现布局。具体的步骤是：先写好各个块的html并给类别来进行标识，在css中严格设置各个块的大小，然后通过float来布局，最后通过每个模块的margin来微调。各大模块的位置布局好后，再进入到每个模块内部，进行细节的实现。
  • 图片画廊下的选择小圆点的实现。一个 div 包裹多个 span，然后将 span 设为 inline-block，而非 设置多个块级元素再布局。

  • 注意一些内联元素的排列，直接连续排列，不要回车隔开，因为浏览器会将内联元素间的换行解析为一个空格。硬要将内联元素换行，还希望内联元素之间没有空格的话，可以在父级设置 font-size = 0。

  • imgBox2 内的图片，用弹性盒模型来进行对齐布局。参考资料：CSS justify-content 属性、CSS align-content 属性。

  • 注意一些样式的继承关系对效果的影响。比如 <a><img></a>，要调整图片的大小，应该调整img标签而非a标签。<div><a>\xxxx</div>，要改变xxxx字体颜色要修改a标签，改字体可以改div标签进而继承到a标签中。参考资料：CSS 层叠与继承。

  • 网页中的小图标可能会集中放在一个图片文件中。一来可以减少浏览器加载页面时对服务器的请求次数，二来用CSS插入各个小图标的时候可以同过 引用相同的图片但使用不同的定位 来实现。

  • text-align 属性。可以对子元素进行对齐，但似乎对块级元素无效。

  • 所有HTML元素都可以被看作盒子 菜鸟-盒子元素。行内元素、块元素：MDN-行内元素、MDN-块元素

  • HTML元素，有的主要是当“容器”（多是块元素，div、p、ul、li等），有的是直接带有“内容”（多是行内元素，span、a、img等）；而且有的是通过父级元素设置样式（子元素继承）来控制内容表现（p元素的font、color、align等），有的则是对内容直接对应的元素设置样式来控制内容表现（a元素的color、img的width、iconfont的font等）。这个点容易搞混。

  • 插入的 iconfont 小图标，需要单独设置样式，父级的字体样式不会继承下去。且 iconfont 小图标的垂直居中，要对iconfont用 verticle-align: middle 或 line-height。

  • 有关行内元素的宽高设置：块级元素和内联元素的宽高是如何确定的。当我们设置侧边浮动导航栏的淘公仔的时候，它在 span 标签下，它的在设置定位前，宽高都是 32（我们设置的font-size）；但将它的定位设置为 position: absolute 后，它的宽高由我们设置的 width 和 height 更变为 50x36。

远程仓库操作

本地仓库的操作以上教程讲的非常清晰详细，但是有关远程仓库的pull和push，模型比较模糊。

  • 在逻辑上，将远程分支dev看作本地分支dev上的一条分支，以此来思考push前是否要pull（远程dev和本地dev合并，是直接合并还是解决冲突），也以此理解pull的合并操作。
  • push最终的效果，就是将本地的一个commit提交到了远程分支上
  • push和merge只看两条分支的末端commit（看是否冲突），并且会生成一个新的commit

以上想法大体是合理的，但更准确的远程仓库和本地仓库的关系模型可见 Learn Git Branching (gitee.io)。大体上来说，就是远程仓库分支，在本地仓库有个与之一一对应的分支。fetch 操作就是将远程分支完整地拉去到本地对应的分支上。

React Router

  • React Router 中文文档

  • React Router 尚硅谷 P74-P78

    理解 SPA

    1. 单页面Web应用（single page web application, SPA）
    2. 整个应用只有一个完整的页面
    3. 点击页面中的链接不会刷新页面，只会做页面的局部更新
    4. 数据都需要通过 ajax 请求获取，并在前端异步展现

    理解路由

    1. 什么是路由

      一个路由就是一个映射关系(key:value)。key为点击后浏览器地址栏切换后的路径, value可能是function或component

    2. 路由分类

    • 后端路由

      理解： value是function, 用来处理客户端提交的请求。

      注册路由： router.get(path, function(req, res))

      工作过程：当node接收到一个请求时, 根据请求路径找到匹配的路由, 调用路由中的函数来处理请求, 返回响应数据

    • 前端路由

      浏览器端路由，value是component，用于展示页面内容。

      注册路由: < Route path=”/test” component={Test}>

      工作过程：当浏览器的path变为/test时, 当前路由组件就会变为Test组件

    前端路由原理 + 基本使用

    基于浏览器的历史记录，即 BOM 的 history 对象。在页面点击按钮或链接，取消其默认事件处理，设置事件处理为调用 history API 对浏览器历史记录进行修改（栈结构，历史记录入栈出栈替换等）+ 给 history 对象添加历史记录变更事件的事件处理函数，让历史记录发生改变时做出相应的前端变更。

    在 react-router-dom 的帮助下，以上过程可封装为：将需要路由功能的组件（导航和展示）用一个 BrowserProuter组件 包裹需要路由功能的组件（通常是整个 < APP /> 组件，可创建多个 < BrowserRouter > 组件，不同路由器下有各自的路由规则）+ 创建设置路由链接（点了改变地址栏和历史记录）+ 注册路由（建立地址栏路径和组件映射关系，地址栏变更的时候渲染对应的组件）

    一般组件和路由组件的区别：语法写法不同不必说；一般组件是自然渲染，而路由组件是路由到它的时候渲染；我们区分存放它们的代码文件的位置（一般是 components 和 pages）；渲染实例化时接受的参数props内容不同，一般组件传入什么接收什么，路由组件包含 history, location, match 等。

Redux

  • Redux 官方文档

    Redux 是什么？

    它有助于首先理解这个“Redux”的东西是什么。它有什么作用？它帮助我解决什么问题？我为什么要使用它？

    Redux 是一个使用叫做“action”的事件来管理和更新应用状态的模式和工具库 它以集中式Store（centralized store）的方式对整个应用中使用的状态进行集中管理，其规则确保状态只能以可预测的方式更新。

    为什么要用 Redux？

    Redux 帮你管理“全局”状态 - 哪些应用程序的许多部分都需要的状态。

    Redux 提供的模式和工具使您更容易理解应用程序中的状态何时、何地、为什么以及如何更新，以及当这些更改发生时您的应用程序逻辑将如何表现. Redux 指导您编写可预测和可测试的代码，这有助于让您确信您的应用程序将按预期工作。

    我什么时候应该使用 Redux？

    Redux 可帮助您处理共享状态的管理，但与任何工具一样，它也有权衡。有更多的概念需要学习，还有更多的代码需要编写。它还为您的代码添加了一些额外代码，并要求您遵循某些限制。这是短期和长期生产力之间的权衡。

    Redux 在以下情况下更有用：

    1. 在应用的大量地方，都存在大量的状态

    2. 应用状态会随着时间的推移而频繁更新

    3. 更新该状态的逻辑可能很复杂

    4. 中型和大型代码量的应用，很多人协同开发

    并非所有应用程序都需要 Redux。 花一些时间思考您正在构建的应用程序类型，并决定哪些工具最能帮助解决您正在处理的问题。

    大概了解了 Redux 的核心概念，需要深入时再按着官方文档的教程一路撸下来

    State 管理

    首先回顾一下 React 中没有 Redux 的，状态更新的情况。

    然而，当我们有多个组件需要共享和使用相同state时，可能会变得很复杂，尤其是当这些组件位于应用程序的不同部分时。有时这可以通过 “提升 state” 到父组件来解决，但这并不总是有效。（React 中的 Context 也是为了解决类似的问题，Context 和 Redux 的对比？）

    解决这个问题的一种方法是从组件中提取共享 state，并将其放入组件树之外的一个集中位置。这样，我们的组件树就变成了一个大“view”（即共享状态改变，共享之的组件的UI表现都会改变），任何组件都可以访问 state 或触发 action，无论它们在树中的哪个位置！（即，此前 View 和 State 都在某一个（父子）组件上，现在将 State 分离出来到另一个地方去了）

    通过定义和分离 state 管理中涉及的概念并强制执行维护 view 和 state 之间独立性的规则，代码变得更结构化和易于维护。

    这就是 Redux 背后的基本思想：应用中使用集中式的全局状态来管理，并明确更新状态的模式，以便让代码具有可预测性。

Hexo+Github+阿里云 搭建个人博客总结

什么是 HEXO

Hexo 是一款基于 Node.js 的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在 GitHub 和 Heroku 上，是搭建博客的首选框架。这里我们选用的是 GitHub。Hexo 同时也是 GitHub 上的开源项目。如果想要更加全面的了解 Hexo，可以到其官网 Hexo 官网 了解更多的细节，因为 Hexo 的创建者是台湾人，对中文的支持很友好，可以选择中文进行查看。

此次我搭建个人博客的技术选型是：Hexo + Github + 阿里云。接下来是具体的步骤。

获得个人网站域名

申请域名的地方有很多，这里推荐 阿里云：阿里云-为了无法计算的价值 申请入口：域名注册 购买域名这也是我们整个搭建过程中惟一一个需要花钱的地方。如果你已经有了空闲域名就无需购买，直接使用即可。

在 Git Bash 中检测 GitHub 公钥设置是否成功，输入 ssh git@github.com ：

如上则说明成功。这里之所以设置 GitHub 密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在 GitHub 上，私钥放置在自己的电脑里。GitHub 要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了 ssh，当你推送的时候，git 就会匹配你的私钥跟 GitHub 上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

踩坑：

  • 梯子有时候不稳定，导致 git clone，hexo d 等操作因为网络问题而失败。多换换梯子节点，多试试就好。
  • hexo d 始终连接不到 github 的话，打开 git bash 输入 ssh -T git@github.com 看看能不能连上，不能的话参考 错误：权限被拒绝（公钥） - GitHub Docs；能的话说明是 hexo 配置的问题，再谷歌找找原因，Hexo部署出现错误err: Error: Spawn failed解决方式。
  • hexo s 在本地已显示出了正常的网页，但 hexo d 到 github 后，xxxx.gihub.com 能访问，但是一片空白。确保此前步骤无误的话，多刷新一下 github 就好了……
  • 本地能正常显示页面，但访问网址不行的时候：检查是否部署成功，检查 github page 设置，检查域名（阿里云）问题
  • 添加 CNAME 文件

找到 hexo 博客根目录下的 source 文件夹。新建一个文件，名为 CNAME ，内容为你的域名（不包括 www.或 http 等等）。不然每次 hexo d 部署后，Github Pages 页面设置的域名会被清空

绑定域名

虽然在 Internet 上可以访问我们的网站，但是网址是 GitHub 提供的:http://xxxx.github.io (知乎排版可能会出现”http://“字样) 而我们想使用我们自己的个性化域名，这就需要绑定我们自己的域名。这里演示的是在阿里云万网的域名绑定，在国内主流的域名代理厂商也就阿里云和腾讯云。登录到阿里云，进入管理控制台的域名列表，找到你的个性化域名，进入解析。
